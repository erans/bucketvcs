## 9. FAQ

### 9.1 Why does `git lfs push` say "object exists, skipping" but the verify still fires?

[The Batch handler returns an empty `actions` map when the object is
present at the claimed size. git-lfs skips the upload PUT but still
calls verify (it's part of its protocol contract). The verify-success
audit event fires regardless. Not a bug.]

### 9.2 Why isn't there a per-repo LFS toggle?

[Per the spec §8: LFS objects only land in storage when a client pushes
them. Per-repo opt-out would add control-plane complexity for no storage
saving. Repos that don't use LFS pay nothing.]

### 9.3 Why does the SSH-authenticate response carry Basic auth, not Bearer?

[Documented in `internal/lfs/auth.go`: the M4 gateway's RunAuth path
only parses Basic. Bearer requires a separate credential type that
isn't shipped in M13. The git-lfs client treats the header opaquely so
Basic works without any client change.]

### 9.4 Where do I find the byte-count of uploaded LFS objects?

[Deliberately not in `lfs_*_total` metrics — see spec §7 note. Use the
backend's own access logs / billing dashboards. On S3, enable bucket
server access logging. On GCS, enable audit logs.]

#### Step 2: Write the script

Create `scripts/m13-lfs-smoke-minio.sh` (chmod 755):

```bash
#!/usr/bin/env bash
# scripts/m13-lfs-smoke-minio.sh
#
# End-to-end smoke for M13 LFS against MinIO (S3-compatible direct-mode
# signed URLs):
#   1. Boot MinIO via docker compose (idempotent).
#   2. Create the test bucket.
#   3. Set up bucketvcs auth, register the repo against s3://bucket
#      (skip M1 init — the import step seeds the manifest).
#   4. Start bucketvcs serve with --lfs and the proxied-url-* flags
#      (LFS proxied URL plumbing is reused for /_lfs/ token signing
#      when the SSH command is used; HTTPS LFS against MinIO goes
#      direct via the S3 presign path).
#   5. git push a small repo with an LFS-tracked binary; assert the
#      git-lfs client trace shows `X-Amz-Signature` in the upload PUT
#      (proves direct upload, not gateway proxy).
#   6. git clone fresh + git lfs pull; assert the trace shows
#      `X-Amz-Signature` in the download GET. Compare bytes.
#   7. Tear down (preserves MinIO if BUCKETVCS_KEEP_EMULATORS=1).
#
# Skips with exit 77 if git-lfs or docker is unavailable.
#
# Requirements: docker compose, git ≥ 2.41, git-lfs ≥ 3.0, Go toolchain.

set -euo pipefail

if ! command -v git-lfs >/dev/null 2>&1; then
    echo "SKIP: git-lfs not on PATH; install from https://git-lfs.com/"
    exit 77
fi
if ! command -v docker >/dev/null 2>&1; then
    echo "SKIP: docker not on PATH"
    exit 77
fi
if ! docker compose version >/dev/null 2>&1; then
    echo "SKIP: docker compose v2 not available"
    exit 77
fi

REPO_ROOT="$(cd "$(dirname "${BASH_SOURCE[0]}")/.." && pwd)"
COMPOSE_FILE="$REPO_ROOT/docker-compose.cloud.yml"
KEEP_UP="${BUCKETVCS_KEEP_EMULATORS:-0}"
BUCKET="bucketvcs-lfs-smoke"

echo "==> Building bucketvcs binary"
BIN="$(mktemp)"
go build -o "$BIN" "$REPO_ROOT/cmd/bucketvcs"

echo "==> Booting MinIO via docker compose"
docker compose -f "$COMPOSE_FILE" up -d --wait minio

SERVE_PID=""
ROOT="$(mktemp -d)"

cleanup() {
    rc=$?
    if [[ -n "$SERVE_PID" ]]; then
        kill "$SERVE_PID" 2>/dev/null || true
        wait "$SERVE_PID" 2>/dev/null || true
    fi
    if [[ "$rc" -eq 0 ]]; then
        rm -rf "$ROOT"
        echo "M13_LFS_MINIO_SMOKE_OK"
    else
        echo "(forensics preserved at $ROOT)"
        if [[ -f "$ROOT/serve.log" ]]; then
            echo "==== serve.log tail ===="
            tail -50 "$ROOT/serve.log" || true
        fi
    fi
    rm -f "$BIN"
    if [[ "$KEEP_UP" != "1" ]]; then
        docker compose -f "$COMPOSE_FILE" down -v >/dev/null 2>&1 || true
    fi
    exit "$rc"
}
trap cleanup EXIT

# Create bucket via mc in a throwaway container.
docker run --rm --network host \
    --entrypoint sh \
    minio/mc:RELEASE.2025-01-17T23-25-50Z \
    -c "
        mc alias set local http://localhost:9000 minioadmin minioadmin >/dev/null
        mc mb --ignore-existing local/$BUCKET >/dev/null
    "

export BUCKETVCS_S3_ENDPOINT=http://127.0.0.1:9000
export BUCKETVCS_S3_REGION=us-east-1
export BUCKETVCS_S3_FORCE_PATH_STYLE=true
export AWS_ACCESS_KEY_ID=minioadmin
export AWS_SECRET_ACCESS_KEY=minioadmin

STORE="s3://$BUCKET"
# Unique tenant per run so MinIO state doesn't bleed across runs.
TENANT="t$(basename "$ROOT" | tr -dc '[:alnum:]' | head -c 8)"
REPO="r"

export BUCKETVCS_AUTH_DB="$ROOT/auth.db"
openssl rand -hex 16 > "$ROOT/signing.key"

"$BIN" user add alice
TOKEN=$("$BIN" token create alice | grep -m1 '^bvts_')
if [[ -z "$TOKEN" ]]; then
    echo "FAIL: could not extract token from 'bucketvcs token create'"
    exit 1
fi
"$BIN" repo register "$TENANT/$REPO" --store "$STORE"
"$BIN" repo grant alice "$TENANT/$REPO" write

PORT=$(python3 -c 'import socket;s=socket.socket();s.bind(("127.0.0.1",0));print(s.getsockname()[1]);s.close()')
BASE_URL="http://127.0.0.1:$PORT"

echo "==> Start bucketvcs serve on $BASE_URL"
"$BIN" serve \
    --addr "127.0.0.1:$PORT" \
    --store "$STORE" \
    --mirror-dir "$ROOT/mirror" \
    --lfs \
    --proxied-url-signing-key "$ROOT/signing.key" \
    --proxied-url-base "$BASE_URL" \
    > "$ROOT/serve.log" 2>&1 &
SERVE_PID=$!

for _ in $(seq 1 50); do
    if curl -sf "$BASE_URL/healthz" >/dev/null 2>&1; then
        break
    fi
    sleep 0.2
done
curl -sf "$BASE_URL/healthz" >/dev/null || { echo "FAIL: serve never ready"; exit 1; }
echo "READY"

# Build a working repo with an LFS-tracked binary.
WORK="$ROOT/repo"
mkdir -p "$WORK"
(
    cd "$WORK"
    git init -q -b main
    git lfs install --local
    git lfs track "*.bin"
    git config user.email smoke@example.com
    git config user.name smoke
    head -c 1048576 /dev/urandom > big.bin
    git add .gitattributes big.bin
    git commit -qm "add lfs object"
    git remote add origin "http://alice:$TOKEN@127.0.0.1:$PORT/$TENANT/$REPO.git"
    GIT_TRACE_CURL=1 git push origin main 2> "$ROOT/trace-push.log"
)

if ! grep -q "X-Amz-Signature" "$ROOT/trace-push.log"; then
    echo "FAIL: push trace did not contain X-Amz-Signature; LFS PUT did not go direct to MinIO"
    grep -iE "lfs|upload|amz" "$ROOT/trace-push.log" | head -30
    exit 1
fi
echo "    PUSH: direct X-Amz-Signature confirmed in upload trace"

# Verify the audit events fired.
grep -q 'event=lfs.batch' "$ROOT/serve.log" || { echo "FAIL: missing lfs.batch"; exit 1; }
grep -q 'event=lfs.verify' "$ROOT/serve.log" || { echo "FAIL: missing lfs.verify"; exit 1; }

# Fresh clone + lfs pull.
CLONE="$ROOT/clone"
GIT_LFS_SKIP_SMUDGE=1 git clone "http://alice:$TOKEN@127.0.0.1:$PORT/$TENANT/$REPO.git" "$CLONE"
(
    cd "$CLONE"
    git lfs install --local
    GIT_TRACE_CURL=1 git lfs pull 2> "$ROOT/trace-pull.log"
)

if ! grep -q "X-Amz-Signature" "$ROOT/trace-pull.log"; then
    echo "FAIL: pull trace did not contain X-Amz-Signature; LFS GET did not go direct to MinIO"
    grep -iE "lfs|download|amz" "$ROOT/trace-pull.log" | head -30
    exit 1
fi
echo "    PULL: direct X-Amz-Signature confirmed in download trace"

cmp "$WORK/big.bin" "$CLONE/big.bin" || { echo "FAIL: cloned bytes differ"; exit 1; }

echo "M13 LFS MinIO smoke: OK"
```

## Scope decisions (committed; not in spec)

- LFS proxied URLs use `LFSProxiedURLSigningKey/LFSProxiedBaseURL`
  independent of the gateway-wide `ProxiedURLSigningKey/ProxiedBaseURL`
  (no `ProxiedKeyResolver` required for LFS). Why: keeps `/_lfs/` from
  forcing `/_bundle/` + `/_pack/` mounts on operators who want only LFS.
- SSH-authenticate response carries **HTTP Basic** Authorization
  (`Basic base64(userName:bvts_...)`) not Bearer. Why: gateway's M4
  RunAuth only parses Basic.
- `IssueSSHToken` rejects `:` and control characters in userName at
  the credential boundary because CreateUser does not enforce them.
- Deploy-key actors (`UserID` prefix `deploy:`) cannot mint LFS
  bearers because their UserID is not a real users.id FK.
- HTTPS LFS works without `--proxied-url-base` (verify URL is built
  from the inbound request Host); SSH LFS requires `--proxied-url-base`
  (no inbound request to derive from). serve.go warns but does not
  fail-start when --proxied-url-base is missing with --lfs.
