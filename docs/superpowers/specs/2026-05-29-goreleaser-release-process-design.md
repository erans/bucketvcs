# GoReleaser-based Tagged Release Process — Design

**Date:** 2026-05-29
**Status:** Approved (design)

## Goal

Replace the hand-rolled release workflow with a [GoReleaser](https://goreleaser.com)-driven
tagged release process that, on a pushed semver tag:

1. Cross-compiles the `bucketvcs` binary for Windows, macOS, and Linux.
2. Builds `.deb` and `.rpm` packages for Linux.
3. Distinguishes **pre-releases** (tags with a `-rcN`/`-beta`/`-alpha` suffix) from
   **full releases** (clean `vX.Y.Z` tags), marking the GitHub Release accordingly.

Also bump the toolchain to the latest Go 1.26 (**go1.26.3**).

## Current State (baseline)

- Go 1.25, pure-Go (`CGO_ENABLED=0`), single binary `cmd/bucketvcs`.
- Version stamped via `-ldflags "-X main.buildVersion=<tag>"` (var at `cmd/bucketvcs/serve.go:41`,
  default `"0.1-dev"`).
- `.github/workflows/release.yml` triggers on `v*` tags. A `test` gate job runs `go vet` +
  `go test`, then a `release` job hand-rolls a bash cross-compile of 5 targets, produces
  `.tar.gz`/`.zip` + `SHA256SUMS.txt`, and publishes via `softprops/action-gh-release@v2`
  with `generate_release_notes: true`.
- **Missing vs. goal:** no `.deb`/`.rpm`; no pre-release distinction.
- Repo-root files available for packaging: `README.md`, `LICENSE`, `NOTICE`,
  `THIRD_PARTY_LICENSES`.

## Design

### 1. Trigger & versioning

- **Tag scheme (semver):** `vX.Y.Z` → full release; `vX.Y.Z-rcN` (or any `-` prerelease suffix)
  → pre-release.
- GoReleaser `release.prerelease: auto` inspects the tag's semver prerelease segment: a
  non-empty suffix (`-rc1`, `-beta`, `-alpha`, …) marks the GitHub Release as a pre-release;
  a clean tag publishes a full release. No conditional branching in the workflow.
- The `on: push: tags: ['v*']` trigger is retained. Milestone tags (`mNN-*`) remain ignored.
- Version is stamped into the existing var via ldflags (see §4 builds). No application code
  change is required — `main.buildVersion` continues to be the single version var.

### 2. Build matrix

Pure-Go cross-compile (`CGO_ENABLED=0`), matching what ships today:

| OS      | Arch          |
| ------- | ------------- |
| linux   | amd64, arm64  |
| darwin  | amd64, arm64  |
| windows | amd64         |

Build flags: `-trimpath`, `-ldflags "-s -w -X main.buildVersion={{ .Version }}"`.

### 3. Artifacts

- **Archives** — `bucketvcs_<version>_<os>_<arch>.tar.gz` (unix) / `.zip` (windows), each
  containing: the binary (`bucketvcs` / `bucketvcs.exe`), `README.md`, `LICENSE`, `NOTICE`,
  `THIRD_PARTY_LICENSES`.
- **Linux packages (nfpm, built into GoReleaser)** — `.deb` and `.rpm` for `amd64` and
  `arm64`. **Binary only:**
  - `/usr/bin/bucketvcs`
  - docs under `/usr/share/doc/bucketvcs/` (`README.md`, `LICENSE`, `NOTICE`,
    `THIRD_PARTY_LICENSES`)
  - No systemd unit, no service user, no pre/post-install scriptlets, no conffiles.
- **`checksums.txt`** — SHA256 over all archives and packages (GoReleaser default name;
  replaces the prior `SHA256SUMS.txt`).
- **Release notes** — auto-generated by GoReleaser from commit history (replaces
  `action-gh-release`'s `generate_release_notes`).

### 4. Workflow changes — `.github/workflows/release.yml`

- **Keep** the `test` gate job (checkout, `setup-go`, git identity config, `go vet`,
  `go test -count=1 ./...`), still a `needs:` dependency of the release job.
- **Replace** the hand-rolled bash build/publish steps in the `release` job with
  `goreleaser/goreleaser-action@v6` running `release --clean`.
  - `permissions: contents: write` retained (create the GitHub Release).
  - `GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}` passed to the action.
- New `.goreleaser.yaml` at repo root holds all build/archive/nfpm/checksum/changelog/release
  config.

### 5. Go 1.26 bump

- `go.mod`: `go 1.25.0` → `go 1.26.0`.
- All `setup-go` workflow steps bumped to Go `1.26.3`: `ci.yml`, `conformance.yml`,
  `conformance-cloud.yml`, `release.yml`.

### 6. Verification

- `goreleaser check` validates `.goreleaser.yaml` (add as a step / run locally).
- `goreleaser release --snapshot --clean` is a dry run that builds archives + `.deb`/`.rpm`
  without publishing — used to prove the config before the first real tag.
- After bumping Go: `go vet ./...` and `go test -count=1 ./...` pass on 1.26.3.

## Out of Scope (YAGNI for this cut)

- Artifact signing (cosign / GPG).
- Homebrew tap, Scoop, Docker/OCI images.
- systemd service unit, service user, config-file packaging.
- Universal macOS binaries / notarization.

## Acceptance Criteria

1. Pushing `vX.Y.Z` produces a **full** GitHub Release with: 5 binary archives, `.deb` +
   `.rpm` for amd64 and arm64, and `checksums.txt`.
2. Pushing `vX.Y.Z-rcN` produces an equivalent set of artifacts on a GitHub Release flagged
   **pre-release**.
3. `bucketvcs` (from any built artifact) reports the tag as its version (via `buildVersion`).
4. `go.mod` declares `go 1.26.0`; all workflows build on Go `1.26.3`; `go vet` + `go test`
   pass.
5. `goreleaser check` passes and a `--snapshot` build succeeds locally.
