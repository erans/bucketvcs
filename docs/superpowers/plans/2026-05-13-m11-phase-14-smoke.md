# M11 Phase 14 — End-to-End Smoke Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship the M11 end-to-end smoke test (MinIO direct-mode signed URLs) and the final M11 verification gate that tags the milestone complete.

**Architecture:** One shell script (`scripts/m11-smoke-minio.sh`) that spins up the existing MinIO emulator from `docker-compose.cloud.yml`, runs a full M11 round-trip (init → import → maintenance with bundle generation → `bucketvcs serve` advertising direct signed URLs → `git clone` consuming the bundle), and asserts both `/_bundle/` and `/_pack/` markers OR `X-Amz-Signature` in the git trace. Then a verification gate that runs the full Go test suite, `go vet`, and both available smoke scripts. The plan's original localfs smoke is intentionally DEFERRED — the gateway in `cmd/bucketvcs/serve.go` does not mount `/_bundle/` or `/_pack/` inbound routes (Phase 8 deferred work; see §Deferred), so a localfs smoke would mint URLs the gateway cannot serve.

**Tech Stack:** Bash. Docker (for MinIO via `docker compose -f docker-compose.cloud.yml`). The bucketvcs CLI built at HEAD. Standard `git` ≥ 2.41 (the host's `git` must support `transfer.bundleURI` and `fetch.uriProtocols`).

## Plan-vs-reality corrections to fold in

Three concrete corrections versus the master plan's Phase 14 prose:

1. **Git config flag.** The master plan said `-c fetch.bundleURI=true`. The actual git config that activates bundle-uri during clone is `transfer.bundleURI=true`. Verified in Phase 10 plan-vs-reality (see `m11_progress`); also matches `internal/diffharness/bundleuri_test.go:273` which uses `transfer.bundleURI=true`.

2. **Strong markers in trace.** The master plan's localfs smoke used `grep -q bundle` on the trace. This is a fail-open assertion — the substring `bundle` appears in argv echo (`def_param: transfer.bundleURI=true`) regardless of server behavior. The MinIO smoke must use:
   - For bundle-uri direct mode: `X-Amz-Signature` in the trace (the signed URL the client downloads), OR the substring `signedurl` in def_param events.
   - For pack-uri direct mode: same `X-Amz-Signature` (or any S3-style signature marker).
   - Negative marker: `--keep=fetch-pack` should NOT appear (its presence indicates the client fell through to standard fetch).

3. **Store URL + credentials format.** The master plan said `--store=s3://bucketvcs-test?endpoint=http://127.0.0.1:9000&access-key=minioadmin&secret-key=minioadmin&region=us-east-1`. The actual bucketvcs CLI does NOT parse credentials from query parameters; it reads:
   - `BUCKETVCS_S3_ENDPOINT` (env var) for the endpoint URL
   - `BUCKETVCS_S3_REGION` (env var) for region
   - `BUCKETVCS_S3_FORCE_PATH_STYLE` (env var; required for MinIO)
   - `AWS_ACCESS_KEY_ID` and `AWS_SECRET_ACCESS_KEY` (standard AWS env vars)

   And the `--store=s3://<bucket>[/<prefix>]` URL carries only the bucket and prefix. See `cmd/bucketvcs/store.go:133-144` and `internal/storage/s3compat/url.go:10`.

4. **CLI positional args.** `bucketvcs init` and `bucketvcs import` take tenant and repo as positional arguments, not as `--repo=t/r`. Specifically:
   - `bucketvcs init [flags] <tenant> <repo>`
   - `bucketvcs import [flags] <source-bare-repo> <tenant> <repo>`
   - `bucketvcs maintenance --repo=<tenant>/<repo>` (this one IS a flag — different ergonomics, but established)

### Step 1: Confirm MinIO emulator is in compose file

### Step 2: Confirm host `git` supports `transfer.bundleURI`

Run:
```bash
git --version
```

Expected: 2.41 or later. The protocol-v2 bundle-uri client landed in 2.41; older clients silently ignore `transfer.bundleURI=true` and the script will fail to find `X-Amz-Signature` in the trace.

### Step 3: Write the script

Create `scripts/m11-smoke-minio.sh` with this exact content:

```bash
#!/usr/bin/env bash
# M11 end-to-end smoke against MinIO (S3-compatible direct-mode signed URLs):
#   1. Boot MinIO via docker compose (idempotent).
#   2. Init a bucketvcs repo + import a tiny git history.
#   3. Run maintenance --force (full pipeline: repack + reachability + bundle).
#   4. Start bucketvcs serve with --bundle-uri-mode=direct + --pack-uri-mode=direct.
#   5. git clone with transfer.bundleURI=true — assert trace contains
#      `X-Amz-Signature` (proves a direct signed URL was advertised).
#   6. git clone with protocol.version=2 + fetch.uriProtocols=https —
#      assert trace contains `X-Amz-Signature` for pack-uri direct serve.
#   7. Tear down (preserves MinIO container if BUCKETVCS_KEEP_EMULATORS=1).
#
# Requirements:
#   - docker compose available
#   - git ≥ 2.41 (transfer.bundleURI support)
#   - Go toolchain available (script uses `go run ./cmd/bucketvcs ...`)
#
# Env var overrides:
#   BUCKETVCS_KEEP_EMULATORS=1   Keep MinIO container running after smoke.

set -euo pipefail

REPO_ROOT="$(cd "$(dirname "${BASH_SOURCE[0]}")/.." && pwd)"
COMPOSE_FILE="$REPO_ROOT/docker-compose.cloud.yml"
KEEP_UP="${BUCKETVCS_KEEP_EMULATORS:-0}"
BUCKET="bucketvcs-smoke"

echo "==> Booting MinIO via docker compose"
docker compose -f "$COMPOSE_FILE" up -d --wait minio

cleanup() {
    rc=$?
    if [[ -n "${SERVE_PID:-}" ]]; then
        kill "$SERVE_PID" 2>/dev/null || true
        wait "$SERVE_PID" 2>/dev/null || true
    fi
    if [[ -n "${ROOT:-}" ]]; then
        rm -rf "$ROOT"
    fi
    if [[ "$KEEP_UP" != "1" ]]; then
        docker compose -f "$COMPOSE_FILE" down -v >/dev/null 2>&1 || true
    fi
    exit "$rc"
}
trap cleanup EXIT

# MinIO bucket creation via mc-in-container (idempotent).
echo "==> Creating MinIO bucket $BUCKET"
docker run --rm --network host \
    --entrypoint sh \
    minio/mc:RELEASE.2025-01-17T23-25-50Z \
    -c "
        mc alias set local http://localhost:9000 minioadmin minioadmin >/dev/null
        mc mb --ignore-existing local/$BUCKET >/dev/null
    "

# Env for bucketvcs CLI to talk to MinIO.
export BUCKETVCS_S3_ENDPOINT=http://127.0.0.1:9000
export BUCKETVCS_S3_REGION=us-east-1
export BUCKETVCS_S3_FORCE_PATH_STYLE=true
export AWS_ACCESS_KEY_ID=minioadmin
export AWS_SECRET_ACCESS_KEY=minioadmin

STORE="s3://$BUCKET"
TENANT="t"
REPO="r"

ROOT="$(mktemp -d)"
KEY="$ROOT/signing-key"
head -c 32 /dev/urandom > "$KEY"

echo "==> Init bucketvcs repo $TENANT/$REPO"
go run ./cmd/bucketvcs init --store="$STORE" "$TENANT" "$REPO"

echo "==> Build a tiny git history and import it"
WORK="$ROOT/work"
git init --quiet -b main "$WORK"
echo "hi" > "$WORK/f"
git -C "$WORK" config user.email t@t
git -C "$WORK" config user.name t
git -C "$WORK" add f
git -C "$WORK" commit --quiet -m init

go run ./cmd/bucketvcs import --store="$STORE" "$WORK" "$TENANT" "$REPO"

echo "==> Run full maintenance (repack + reachability + bundle)"
go run ./cmd/bucketvcs maintenance --store="$STORE" --repo="$TENANT/$REPO" --force

# Pick a free-ish port for serve.
PORT=$(( 30000 + (RANDOM % 20000) ))

echo "==> Start bucketvcs serve on 127.0.0.1:$PORT (direct mode)"
go run ./cmd/bucketvcs serve \
    --store="$STORE" \
    --addr="127.0.0.1:$PORT" \
    --bundle-uri-mode=direct \
    --pack-uri-mode=direct \
    --proxied-url-signing-key="$KEY" \
    --proxied-url-base="http://127.0.0.1:$PORT" &
SERVE_PID=$!

# Poll until serve responds, up to 30s.
for i in $(seq 1 30); do
    if curl -sf "http://127.0.0.1:$PORT/$TENANT/$REPO.git/info/refs?service=git-upload-pack" >/dev/null 2>&1; then
        break
    fi
    sleep 1
done

CLONE_BUNDLE="$ROOT/clone-bundle"
CLONE_PACK="$ROOT/clone-pack"

echo "==> Test 1: git clone with transfer.bundleURI=true (expect bundle-uri direct)"
GIT_TRACE2=1 git \
    -c protocol.version=2 \
    -c transfer.bundleURI=true \
    clone --quiet "http://127.0.0.1:$PORT/$TENANT/$REPO.git" "$CLONE_BUNDLE" \
    2> "$ROOT/trace-bundle.log"

if ! grep -q "X-Amz-Signature" "$ROOT/trace-bundle.log"; then
    echo "FAIL: bundle-uri clone trace did not contain X-Amz-Signature"
    echo "----- trace excerpt -----"
    grep -i "bundle\|signature\|amz" "$ROOT/trace-bundle.log" | head -40
    exit 1
fi
if grep -q "\-\-keep=fetch-pack" "$ROOT/trace-bundle.log"; then
    echo "FAIL: bundle-uri clone fell through to standard fetch (--keep=fetch-pack seen)"
    exit 1
fi
echo "    bundle-uri direct signed URL confirmed (X-Amz-Signature present)"

echo "==> Test 2: git clone with fetch.uriProtocols=https (expect packfile-uri direct)"
GIT_TRACE2=1 git \
    -c protocol.version=2 \
    -c fetch.uriProtocols=https \
    clone --quiet "http://127.0.0.1:$PORT/$TENANT/$REPO.git" "$CLONE_PACK" \
    2> "$ROOT/trace-pack.log"

if ! grep -q "X-Amz-Signature" "$ROOT/trace-pack.log"; then
    echo "FAIL: packfile-uri clone trace did not contain X-Amz-Signature"
    echo "----- trace excerpt -----"
    grep -i "packfile\|signature\|amz" "$ROOT/trace-pack.log" | head -40
    exit 1
fi
echo "    packfile-uri direct signed URL confirmed (X-Amz-Signature present)"

# Sanity: cloned heads match the source HEAD.
SRC_HEAD=$(git -C "$WORK" rev-parse HEAD)
B_HEAD=$(git -C "$CLONE_BUNDLE" rev-parse HEAD)
P_HEAD=$(git -C "$CLONE_PACK" rev-parse HEAD)
if [[ "$SRC_HEAD" != "$B_HEAD" ]] || [[ "$SRC_HEAD" != "$P_HEAD" ]]; then
    echo "FAIL: clone HEAD mismatch (src=$SRC_HEAD bundle-clone=$B_HEAD pack-clone=$P_HEAD)"
    exit 1
fi

echo "M11 MinIO smoke: OK"
```

### Step 4: Make the script executable

Run: `chmod +x scripts/m11-smoke-minio.sh`

### Step 5: Run the smoke

Run: `bash scripts/m11-smoke-minio.sh`

Expected: prints `M11 MinIO smoke: OK` at the end, with `==> ...` progress lines for each step.

If the script fails:
- "FAIL: bundle-uri clone trace did not contain X-Amz-Signature" → either the gateway didn't advertise a direct URL (check `--bundle-uri-mode=direct` was honored — could regress if a future change defaults the mode), or the client doesn't support `transfer.bundleURI` (check git version ≥ 2.41).
- "FAIL: packfile-uri clone trace did not contain X-Amz-Signature" → same diagnosis but for the pack-uri path; verify `PackChecksum` is set on the manifest after maintenance with `bucketvcs inspect-manifest --json | jq '.Packs[].PackChecksum'`.
- "FAIL: clone HEAD mismatch" → suggests the clone succeeded but pulled the wrong content; likely a state-leak from a prior run (check that the bucket is fresh).
- Docker errors → run `docker compose -f docker-compose.cloud.yml ps` to see if MinIO actually came up.
