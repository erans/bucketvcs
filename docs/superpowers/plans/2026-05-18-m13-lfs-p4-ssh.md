# M13 Phase 4 — SSH `git-lfs-authenticate` Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Let a stock `git-lfs` client running over SSH obtain a short-TTL HTTPS bearer via `ssh git@host git-lfs-authenticate <owner>/<repo> <upload|download>` and use that bearer to drive the HTTPS Batch + transfer flow shipped in P0–P3.

**Architecture:** New `internal/lfs/auth.go` exports `IssueSSHToken(ctx, issuer, user, repo, op, ttl) (SSHAuthResponse, error)` and the `TokenIssuer` seam. `internal/sshd` parses the new exec command alongside `git-upload-pack`/`git-receive-pack`, authorizes (matching `op` to `ActionRead`/`ActionWrite`), invokes `IssueSSHToken`, and writes the LFS-spec JSON to stdout. The serve CLI gains `--lfs-ssh-token-ttl` (default 15m).

**Tech Stack:** Go, standard library only (encoding/json, time). Reuses `internal/auth` token primitives (`GenerateToken`, `HashSecret`) and the sqlitestore `CreateToken` method via the new `TokenIssuer` interface.

```go
package lfs

import (
	"context"
	"fmt"
	"time"

	"github.com/bucketvcs/bucketvcs/internal/auth"
)

// SSHAuthResponse is the LFS-spec JSON body returned by the SSH
// `git-lfs-authenticate` command. The client uses Header as-is on the
// HTTPS Batch + transfer calls.
type SSHAuthResponse struct {
	Href      string            `json:"href"`
	Header    map[string]string `json:"header"`
	ExpiresAt time.Time         `json:"expires_at"`
}

// TokenIssuer is the narrow seam IssueSSHToken needs to persist a token
// row. The sqlitestore *Store satisfies it by virtue of its existing
// CreateToken method; tests stub it directly. We deliberately do NOT
// reuse auth.Store — token creation is admin-only and not part of the
// gateway request-path interface.
type TokenIssuer interface {
	CreateToken(ctx context.Context, tokenID, userID, secretHash, label string, expiresAt *int64) error
}

// IssueSSHToken mints a short-TTL HTTP bearer scoped to (userID, repo,
// op) and returns the LFS-spec SSHAuthResponse for that bearer.
//
// op MUST be "upload" or "download". ttl MUST be > 0 (we do NOT impose
// an upper bound here — the caller decides what the operator-facing
// flag accepts).
//
// baseURL is the gateway's external base URL (e.g. https://gw.example).
// The returned Href is "<baseURL>/<userID-tenant>/<repo>.git/info/lfs"
// — wait, NO: the spec uses owner/repo from the request, not userID.
// The Href passes <tenant>/<repo>.git so the client constructs the
// Batch URL from it.
func IssueSSHToken(ctx context.Context, issuer TokenIssuer, userID, tenant, repo, op, baseURL string, ttl time.Duration) (SSHAuthResponse, error) {
	switch op {
	case "upload", "download":
	default:
		return SSHAuthResponse{}, fmt.Errorf("lfs.IssueSSHToken: invalid op %q (want upload or download)", op)
	}
	if ttl <= 0 {
		return SSHAuthResponse{}, fmt.Errorf("lfs.IssueSSHToken: ttl must be > 0 (got %s)", ttl)
	}
	if userID == "" || tenant == "" || repo == "" || baseURL == "" {
		return SSHAuthResponse{}, fmt.Errorf("lfs.IssueSSHToken: userID, tenant, repo, baseURL must be non-empty")
	}

	token, id, secret, err := auth.GenerateToken()
	if err != nil {
		return SSHAuthResponse{}, fmt.Errorf("lfs.IssueSSHToken: GenerateToken: %w", err)
	}
	hash, err := auth.HashSecret(secret)
	if err != nil {
		return SSHAuthResponse{}, fmt.Errorf("lfs.IssueSSHToken: HashSecret: %w", err)
	}

	expires := time.Now().Add(ttl)
	expUnix := expires.Unix()
	label := fmt.Sprintf("lfs-ssh:%s %s/%s", op, tenant, repo)

	if err := issuer.CreateToken(ctx, id, userID, hash, label, &expUnix); err != nil {
		return SSHAuthResponse{}, fmt.Errorf("lfs.IssueSSHToken: CreateToken: %w", err)
	}

	return SSHAuthResponse{
		Href:      fmt.Sprintf("%s/%s/%s.git/info/lfs", baseURL, tenant, repo),
		Header:    map[string]string{"Authorization": "Bearer " + token},
		ExpiresAt: expires,
	}, nil
}
```

Note: the function signature has 7 args (issuer, userID, tenant, repo, op, baseURL, ttl), not 5 as the spec sketch suggested. The two extra args (tenant + baseURL) are necessary — the spec sketch was illustrative, not literal. Update the test accordingly: the test above already uses the 7-arg form.

```go
func ParseExecCommand(s string) (*ExecCommand, error) {
	if s == "" {
		return nil, errors.New("empty command")
	}
	verb, rest, hasArg := strings.Cut(s, " ")
	if !hasArg || rest == "" {
		return nil, errors.New("command requires a path argument")
	}

	switch verb {
	case "git-upload-pack", "git-receive-pack":
		op := OpUpload
		if verb == "git-receive-pack" {
			op = OpReceive
		}
		arg, err := stripQuotes(rest)
		if err != nil {
			return nil, err
		}
		if strings.HasPrefix(arg, "/") {
			arg = arg[1:]
		}
		tenant, repo, err := splitRepoPath(arg)
		if err != nil {
			return nil, err
		}
		return &ExecCommand{Op: op, Tenant: tenant, Repo: repo}, nil

	case "git-lfs-authenticate":
		// Expect exactly two whitespace-separated args: <path> <op>.
		// We do NOT support quoted paths here — git-lfs never quotes
		// the path on this command, and accepting quoting would
		// complicate the two-arg split.
		fields := strings.Fields(rest)
		if len(fields) != 2 {
			return nil, errors.New("git-lfs-authenticate requires exactly <owner>/<repo> <upload|download>")
		}
		path, lfsOp := fields[0], fields[1]
		if lfsOp != "upload" && lfsOp != "download" {
			return nil, errors.New("git-lfs-authenticate op must be upload or download")
		}
		// Accept both with and without a trailing .git suffix.
		if strings.HasPrefix(path, "/") {
			path = path[1:]
		}
		if !strings.HasSuffix(path, ".git") {
			path += ".git"
		}
		tenant, repo, err := splitRepoPath(path)
		if err != nil {
			return nil, err
		}
		return &ExecCommand{Op: OpLFSAuthenticate, Tenant: tenant, Repo: repo, LFSOp: lfsOp}, nil

	default:
		return nil, errors.New("command not allowed")
	}
}
```

```go
func (s *Server) handleLFSAuthenticate(ctx context.Context, cmd *ExecCommand, actor *auth.Actor, ch ssh.Channel, perm auth.Perm, flags auth.RepoFlags) error {
	logger := s.opts.Logger
	repoFQN := cmd.Tenant + "/" + cmd.Repo
	user := ""
	if actor != nil {
		user = actor.Name
	}

	if s.opts.LFSTokenIssuer == nil || s.opts.LFSBaseURL == "" {
		emitLFSSSHAuthMetric(ctx, logger, cmd.LFSOp, "disabled")
		emitLFSSSHAuth(ctx, logger, repoFQN, user, cmd.LFSOp, 0, "disabled")
		sendStderrLine(ch, "bucketvcs: lfs not enabled on this gateway")
		sendExitStatus(ch, 1)
		return nil
	}

	// Refine the auth.Action using cmd.LFSOp (ExecOp.RequiredAction
	// fail-closes to Write; download only needs Read).
	required := auth.ActionWrite
	if cmd.LFSOp == "download" {
		required = auth.ActionRead
	}
	ok, _ := auth.Decide(actor, perm, required, flags)
	if !ok {
		emitLFSSSHAuthMetric(ctx, logger, cmd.LFSOp, "forbidden")
		emitLFSSSHAuth(ctx, logger, repoFQN, user, cmd.LFSOp, 0, "forbidden")
		sendStderrLine(ch, "bucketvcs: insufficient permissions")
		sendExitStatus(ch, 128)
		return nil
	}

	// Anonymous actors cannot mint tokens (no user_id to scope them to).
	if actor == nil || actor.ID == "" {
		emitLFSSSHAuthMetric(ctx, logger, cmd.LFSOp, "anon")
		emitLFSSSHAuth(ctx, logger, repoFQN, user, cmd.LFSOp, 0, "anon")
		sendStderrLine(ch, "bucketvcs: anonymous keys cannot mint LFS bearers")
		sendExitStatus(ch, 128)
		return nil
	}

	ttl := s.opts.LFSSSHTokenTTL
	if ttl <= 0 {
		ttl = 15 * time.Minute
	}
	resp, err := lfs.IssueSSHToken(ctx, s.opts.LFSTokenIssuer, actor.ID, cmd.Tenant, cmd.Repo, cmd.LFSOp, s.opts.LFSBaseURL, ttl)
	if err != nil {
		emitLFSSSHAuthMetric(ctx, logger, cmd.LFSOp, "error")
		emitLFSSSHAuth(ctx, logger, repoFQN, user, cmd.LFSOp, int64(ttl/time.Second), "error")
		sendStderrLine(ch, "bucketvcs: failed to issue lfs token")
		sendExitStatus(ch, 1)
		return nil
	}

	body, err := json.Marshal(resp)
	if err != nil {
		sendStderrLine(ch, "bucketvcs: marshal failed")
		sendExitStatus(ch, 1)
		return nil
	}
	if _, err := ch.Write(append(body, '\n')); err != nil {
		return err
	}
	emitLFSSSHAuthMetric(ctx, logger, cmd.LFSOp, "ok")
	emitLFSSSHAuth(ctx, logger, repoFQN, user, cmd.LFSOp, int64(ttl/time.Second), "ok")
	sendExitStatus(ch, 0)
	return nil
}
```

## Cross-task notes

- `OpLFSAuthenticate.RequiredAction()` returns Write so unconfigured callers (e.g. someone reading the enum out of context) fail closed. The session dispatcher overrides this using `cmd.LFSOp` BEFORE invoking `auth.Decide`. Tests assert both.
- The `TokenIssuer` interface is narrower than `auth.Store` on purpose: gateway request-path callers should never reach token-creation methods. `auth.Store` stays read-only from the gateway's perspective.
- `LFSBaseURL` is the same external URL operators already supply via `--proxied-url-base`. We deliberately do not introduce a separate `--lfs-base-url` flag — one external URL is enough.
- HTTPS LFS over `requestBaseURL` (from P3) continues to work without `--proxied-url-base`; only the SSH-issued bearer path requires the operator to configure an external URL (because there is no inbound request to derive it from).
- Phase 4 does NOT introduce a verify-token mechanism; the `Authorization`-echo SECURITY note in `internal/lfs/handler.go` remains. That mitigation is deferred to a follow-on milestone.
